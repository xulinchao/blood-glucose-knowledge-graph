# TraeWork 工作流说明

TraeWork 与 Cursor 共用本仓库：**同一数据、同一真实性规则**。  
规则全文见根目录 [AGENTS.md](../AGENTS.md)「选题与内容真实性」；选题/写稿任务另见 [.cursor/skills/blood-glucose-topic-authenticity/SKILL.md](../.cursor/skills/blood-glucose-topic-authenticity/SKILL.md)。

维护者**非执业医生** — TraeWork 生成的一切文案均须遵守对抗性审核与口播底线。

---

## 两条流水线

```
┌─────────────────────────────────────────────────────────────┐
│  任务 A：每日采集（无需 LLM，Windows 计划任务 / TraeWork）      │
│  scripts/run-daily-harvest.bat                                │
│    → data/exports/daily-harvest-YYYY-MM-DD.json              │
│    → research-daily/YYYY-MM-DD-采集日报.html                  │
│    → data/discovered-topics.json（仅 topic_pick 条目）         │
└─────────────────────────────────────────────────────────────┘
                              ↓ 人工或 TraeWork 读取
┌─────────────────────────────────────────────────────────────┐
│  任务 B：监控日报（TraeWork Agent，可选）                       │
│  输入：采集日报 JSON + 权威来源链接                             │
│  输出：research-daily/YYYY-MM-DD-日报.html                    │
│  侧重：核实、选题 A/B 脚本建议、待核实清单                      │
└─────────────────────────────────────────────────────────────┘
```

| | 采集日报 | 监控日报 |
|---|----------|----------|
| 谁生成 | Python 脚本 | TraeWork Agent |
| 文件名 | `*-采集日报.html` | `*-日报.html` |
| 真实性 | 脚本内对抗性评分 | Agent 必须二次核实 |
| 可否当医学结论 | 否（含 hook_only） | 仅核实后可写「已核实」 |

## TraeWork 定时任务配置

### 任务 1：每日采集（推荐 08:00）

- **类型**：运行程序 / Shell
- **命令**：
  ```bat
  I:\Project\血糖知识图谱\scripts\run-daily-harvest.bat
  ```
- **前提**：Chrome 开、小红书/x.com/reddit 已登录（OpenCLI 扩展已连接）
- **产物**：打开 `research-daily/YYYY-MM-DD-采集日报.html` 查看

### 任务 2：监控日报（推荐 09:00 或手动）

将下方「Agent 任务提示词」整段粘贴到 TraeWork 任务说明。

## TraeWork Agent 任务提示词（监控日报）

```markdown
# 角色
你是血糖/控糖科普编辑，非医生。工作目录为本仓库根目录。

# 必读
1. AGENTS.md「选题与内容真实性」— 第一性原理 + 对抗性审核
2. 当日 data/exports/daily-harvest-YYYY-MM-DD.json
3. 当日 research-daily/YYYY-MM-DD-采集日报.md（若有）

# 任务
基于采集结果写 research-daily/YYYY-MM-DD-日报.html（保留 <!-- Generated by Trae Work -->）。

# 真实性（对抗性审核，必做）
对每条要写入日报的主张：
- 问：若作者只想骗点击，这条标题在暗示什么？
- 无 A/B 级来源（nhc.gov.cn、dxy、期刊）的数字/疗效 → 标「待核实」
- evidence_tier D/E 或 use_as=hook_only → 只写「网友/UP主在讨论 X」，不写为事实
- 禁止：根治、保证逆转、替医嘱用药

# 结构
1. 今日热点（2–4 条，附来源链接与核实状态 badge）
2. 趋势观察
3. 政策/指南动态（仅 A/B 来源）
4. 可转化口播选题 Top 2（注明 evidence_tier、需核实项）
5. 待核实清单（TraeWork 未完成的 fact-check）

# 口播底线（摘要或选题区须出现）
个体情况请咨询医生，不构成医疗建议。
```

## 常见问题

**Q：TraeWork 还要自己搜一遍吗？**  
A：监控日报不必重复跑采集脚本；在采集结果上做**解读与核实**。缺平台时再调 Agent Reach。

**Q：采集失败？**  
A：看采集日报「采集异常」；常见为 OpenCLI 未连接、未登录小红书/X。

**Q：和 Cursor 会冲突吗？**  
A：不会。共用 `discovered-topics.json`，去重由脚本 `dedup_key` 处理；两边都读 `AGENTS.md`。
