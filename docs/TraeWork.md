# TraeWork 工作流说明

TraeWork 与 Cursor 共用本仓库：**同一数据、同一真实性规则**。  
规则全文见根目录 [AGENTS.md](../AGENTS.md)「选题与内容真实性」。

| Skill | 用途 |
|-------|------|
| [.cursor/skills/blood-glucose-daily-brief/SKILL.md](../.cursor/skills/blood-glucose-daily-brief/SKILL.md) | **查今日精选**（默认入口，AIHOT 式） |
| [.cursor/skills/blood-glucose-topic-authenticity/SKILL.md](../.cursor/skills/blood-glucose-topic-authenticity/SKILL.md) | 写稿 / 真实性 / 对抗性审核 |

维护者**非执业医生** — TraeWork 生成的一切文案均须遵守对抗性审核与口播底线。

## 三条流水线

```
┌─────────────────────────────────────────────────────────────┐
│  任务 A：每日采集（无需 LLM，Windows 计划任务 / TraeWork）      │
│  scripts/run-daily-harvest.bat                                │
│    → data/exports/daily-harvest-YYYY-MM-DD.json   （全量）     │
│    → data/exports/daily-brief-YYYY-MM-DD.json     （精选）     │
│    → research-daily/YYYY-MM-DD-采集日报.html                  │
│    → research-daily/YYYY-MM-DD-精选日报.html    ★ Agent 默认读  │
│    → data/discovered-topics.json（仅 gate.passed + topic_pick） │
└─────────────────────────────────────────────────────────────┘
                              ↓ claim_gate 已内置于采集
┌─────────────────────────────────────────────────────────────┐
│  Risk Gate — src/claim_gate.py（规则主张拆解 + 可写性否决）     │
│  heat_score 仅传播 · writability_score / gate.passed 决定可写稿   │
└─────────────────────────────────────────────────────────────┘
                              ↓ Agent 默认读精选 JSON
┌─────────────────────────────────────────────────────────────┐
│  精选层 Schema 2.0 — curate_daily_brief.py                     │
│  时间线按 gate.passed 排序 · 三段式字段 · 理由链 agent_hints      │
│  指针：data/exports/latest-brief.json                         │
└─────────────────────────────────────────────────────────────┘
                              ↓ 可选，需 LLM
┌─────────────────────────────────────────────────────────────┐
│  任务 B：监控日报（TraeWork Agent，可选）                       │
│  输入：精选 JSON + harvest JSON + 权威来源核实                  │
│  输出：research-daily/YYYY-MM-DD-日报.html                    │
│  侧重：政策/指南核实、Top2 口播建议、待核实清单                 │
└─────────────────────────────────────────────────────────────┘
```

| | 采集日报 | **精选日报** | 监控日报 |
|---|----------|--------------|----------|
| 谁生成 | Python | Python（`curate_daily_brief`） | TraeWork Agent |
| **展示（HTML）** | `research-daily/*-采集日报.html` | `research-daily/*-精选日报.html` | `research-daily/*-日报.html` |
| **文档（MD）** | `research-daily/*-采集日报.md`（Agent 用） | `research-daily/*-精选日报.md`（Agent 用） | **`content/research-daily/*-日报.md`**（Quartz 笔记，不在 research-daily 放 MD） |
| 条数 | 全量（100+） | 精选（~12 时间线） | 人工解读 2–4 热点 |
| Agent 默认 | 否 | **是** | 仅「要核实版」时读 HTML |
| 可否当医学结论 | 否 | 否（含 hook 分区） | 仅核实后可写「已核实」 |

## TraeWork 定时任务配置

### 任务 1：每日采集（推荐 08:00）

- **类型**：运行程序 / Shell
- **命令**：
  ```bat
  I:\Project\血糖知识图谱\scripts\run-daily-harvest.bat
  ```
- **前提**：Chrome 开、小红书/x.com/reddit 已登录（OpenCLI 扩展已连接）
- **产物**：
  - 人读：`research-daily/YYYY-MM-DD-精选日报.html`（**先看这个**）
  - 全量：`research-daily/YYYY-MM-DD-采集日报.html`
  - Agent：`data/exports/daily-brief-YYYY-MM-DD.json`

### 任务 2：监控日报（推荐 09:00 或手动）

将下方「Agent 任务提示词」整段粘贴到 TraeWork 任务说明。

---

## TraeWork Agent 任务提示词（监控日报）

```markdown
# 角色
你是血糖/控糖科普编辑，非医生。工作目录为本仓库根目录。

# 必读
1. AGENTS.md「选题与内容真实性」
2. .cursor/skills/blood-glucose-daily-brief/SKILL.md（先读精选）
3. 当日 data/exports/daily-brief-YYYY-MM-DD.json
4. 需要细节时再读 data/exports/daily-harvest-YYYY-MM-DD.json

# 任务
基于**精选日报**产出两份监控日报（不要重复跑采集）：

1. **展示用 HTML**（唯一放在 research-daily）  
   `research-daily/YYYY-MM-DD-日报.html`  
   首行保留 `<!-- Generated by Trae Work -->`

2. **文档用 Markdown**（发 Quartz 笔记）  
   `content/research-daily/YYYY-MM-DD-日报.md`  
   带 YAML frontmatter（title / date / tags），正文与 HTML 同内容、可略精简  
   **不要**在 `research-daily/` 下再写 `*-日报.md`

# 真实性（对抗性审核 + Gate，必做）
对每条要写入日报的主张：
- 问：若作者只想骗点击，这条标题在暗示什么？
- 读精选 JSON 的 `gate` / `writing_safety`；`gate.passed=false` 不得写入可写稿区
- 无 A/B 级来源（nhc.gov.cn、dxy、期刊）的数字/疗效 → 标「待核实」
- evidence_tier D/E 或 use_as=hook_only → 只写「网友/UP主在讨论 X」，不写为事实
- 禁止：根治、保证逆转、替医嘱用药

# 结构
1. 今日热点（2–4 条，优先 gate.passed 的 timeline，附来源与核实 badge）
2. 趋势观察（可引用 cross_platform）
3. 政策/指南动态（仅 A/B 来源，可补充检索）
4. 可转化口播选题 Top 2（仅 writable_picks；每条须含 why_selected、misleading_risk、safe_rewrite）
5. 待核实清单（合并 agent_hints.verify_list + 你未完成的 fact-check）

# 口播底线（摘要或选题区须出现）
个体情况请咨询医生，不构成医疗建议。
```

## 常见问题

**Q：Agent 该读采集还是精选？**  
A：**默认精选**（`latest-brief.json`）。只有用户要「全部条目」才读 harvest 全量。

**Q：TraeWork 还要自己搜一遍吗？**  
A：监控日报在精选基础上**解读与核实**即可；缺 A/B 来源时再 Agent Reach。

**Q：采集失败？**  
A：看采集日报「采集异常」；常见为 OpenCLI 未连接、未登录小红书/X。

**Q：和 Cursor 会冲突吗？**  
A：不会。共用 `discovered-topics.json`，去重由 `dedup_key` 处理。
